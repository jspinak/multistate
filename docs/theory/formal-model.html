Formal Model of MultiState

Overview​

The MultiState framework is built on a rigorous mathematical foundation that extends traditional state machine theory with multi-state activation, dynamic transitions, and occlusion semantics.

Core Definition​

The MultiState system is formally defined as a tuple:

Ω=(S,T,G,ω,Td)\Omega = (S, T, G, \omega, T_d)Ω=(S,T,G,ω,Td​)

Where:

  • SSS is a finite set of states
  • T⊆S×P(S)×P(S)T \subseteq S \times \mathcal{P}(S) \times \mathcal{P}(S)T⊆S×P(S)×P(S) is the transition relation
  • G⊆P(S)G \subseteq \mathcal{P}(S)G⊆P(S) is a set of state groups
  • ω:S×S×R→[0,1]\omega: S \times S \times \mathbb{R} \to [0,1]ω:S×S×R→[0,1] is the occlusion function
  • Td:R×P(S)→P(T)T_d: \mathbb{R} \times \mathcal{P}(S) \to \mathcal{P}(T)Td​:R×P(S)→P(T) is the dynamic transition generator

States​

Definition​

A state s∈Ss \in Ss∈S represents a distinguishable configuration of the system.

Properties​

Each state has:

  • Identity: Unique identifier
  • Elements: Set of UI elements or components
  • Blocking: Boolean indicating if state blocks others
  • Group: Optional membership in state groups

Active States​

At any time ttt, the system maintains a set of active states:

SΞ(t)⊆SS_{\Xi}(t) \subseteq SSΞ​(t)⊆S

Unlike traditional FSMs where ∣SΞ∣=1|S_{\Xi}| = 1∣SΞ​∣=1, MultiState allows ∣SΞ∣≥0|S_{\Xi}| \geq 0∣SΞ​∣≥0.

Transitions​

Definition​

A transition τ∈T\tau \in Tτ∈T is defined as:

τ=(F,A,E)\tau = (F, A, E)τ=(F,A,E)

Where:

  • F⊆SF \subseteq SF⊆S are the "from" states (preconditions)
  • A⊆SA \subseteq SA⊆S are states to activate
  • E⊆SE \subseteq SE⊆S are states to exit

Multi-State Activation​

The key innovation: ∣A∣≥1|A| \geq 1∣A∣≥1, allowing atomic activation of multiple states.

Transition Execution​

Transition τ\tauτ is enabled when:

F⊆SΞ∧(A∩E=∅)F \subseteq S_{\Xi} \land (A \cap E = \emptyset)F⊆SΞ​∧(A∩E=∅)

State Groups​

Definition​

A group g∈Gg \in Gg∈G is a subset of states that activate/deactivate atomically:

g⊆Sg \subseteq Sg⊆S

Group Constraints​

For any transition τ\tauτ and group ggg:

∀s1,s2∈g:(s1∈A(τ)  ⟺  s2∈A(τ))\forall s_1, s_2 \in g: (s_1 \in A(\tau) \iff s_2 \in A(\tau))∀s1​,s2​∈g:(s1​∈A(τ)⟺s2​∈A(τ))

States in the same group must transition together.

Occlusion Function​

Definition​

The occlusion function models state visibility:

ω(sc,sh,t)=p\omega(s_c, s_h, t) = pω(sc​,sh​,t)=p

Where:

  • scs_csc​ is the covering state
  • shs_hsh​ is the hidden state
  • ttt is the current time
  • p∈[0,1]p \in [0,1]p∈[0,1] is the occlusion probability

Occlusion Types​

  1. Modal (p=1.0p = 1.0p=1.0): Complete occlusion
  2. Spatial (p∈[0.5,0.9]p \in [0.5, 0.9]p∈[0.5,0.9]): Partial overlap
  3. Logical (p∈[0.7,1.0]p \in [0.7, 1.0]p∈[0.7,1.0]): Semantic exclusion

Dynamic Transitions​

Definition​

Dynamic transitions are generated at runtime:

Td(t,SΞ)={τ1,τ2,...,τn}T_d(t, S_{\Xi}) = \{\tau_1, \tau_2, ..., \tau_n\}Td​(t,SΞ​)={τ1​,τ2​,...,τn​}

Types​

  1. Reveal Transitions: Generated when occluding state closes
  2. Self-Transitions: Validate current state without change
  3. Discovery Transitions: Found through runtime exploration

Reveal Transition Generation​

When state scs_csc​ covering states H={sh1,sh2,...}H = \{s_h1, s_h2, ...\}H={sh​1,sh​2,...} is removed:

τreveal=({sc},H,{sc})\tau_{reveal} = (\{s_c\}, H, \{s_c\})τreveal​=({sc​},H,{sc​})

Phased Execution Model​

Execution Phases​

Transition execution follows strict phases:

VALIDATE→OUTGOING→ACTIVATE→INCOMING→EXIT\text{VALIDATE} \to \text{OUTGOING} \to \text{ACTIVATE} \to \text{INCOMING} \to \text{EXIT}VALIDATE→OUTGOING→ACTIVATE→INCOMING→EXIT

Phase Definitions​

1. VALIDATE​

Check preconditions: valid(τ)=F(τ)⊆SΞ\text{valid}(\tau) = F(\tau) \subseteq S_{\Xi}valid(τ)=F(τ)⊆SΞ​

2. OUTGOING​

Execute exit actions for current states: outgoing(SΞ)=⋀s∈SΞexit(s)\text{outgoing}(S_{\Xi}) = \bigwedge_{s \in S_{\Xi}} \text{exit}(s)outgoing(SΞ​)=⋀s∈SΞ​​exit(s)

3. ACTIVATE​

Add states to active set (infallible): SΞ′=SΞ∪A(τ)S_{\Xi}' = S_{\Xi} \cup A(\tau)SΞ′​=SΞ​∪A(τ)

4. INCOMING​

Execute entry actions for each activated state: incoming(A)=⋀s∈Aentry(s)\text{incoming}(A) = \bigwedge_{s \in A} \text{entry}(s)incoming(A)=⋀s∈A​entry(s)

5. EXIT​

Remove exited states: SΞ′′=SΞ′∖E(τ)S_{\Xi}'' = S_{\Xi}' \setminus E(\tau)SΞ′′​=SΞ′​∖E(τ)

Rollback Semantics​

If INCOMING fails, rollback ACTIVATE: SΞ,rollback=SΞS_{\Xi,rollback} = S_{\Xi}SΞ,rollback​=SΞ​

Multi-Target Pathfinding​

Problem Definition​

Given current states ScS_cSc​ and target states TTT, find path π\piπ such that:

execute(π,Sc)⊇T\text{execute}(\pi, S_c) \supseteq Texecute(π,Sc​)⊇T

Optimality​

Path π∗\pi^*π∗ is optimal if:

π∗=arg⁡min⁡πcost(π) s.t. T⊆reachable(π,Sc)\pi^* = \arg\min_{\pi} \text{cost}(\pi) \text{ s.t. } T \subseteq \text{reachable}(\pi, S_c)π∗=argminπ​cost(π) s.t. T⊆reachable(π,Sc​)

Complexity​

The search space has complexity:

O(∣V∣⋅2∣T∣)O(|V| \cdot 2^{|T|})O(∣V∣⋅2∣T∣)

Where ∣V∣|V|∣V∣ is the number of states and ∣T∣|T|∣T∣ is the number of targets.

Theorems​

Theorem 1: Activation Atomicity​

Statement: State activation is atomic and infallible.

Proof: Activation is defined as SΞ′=SΞ∪A(τ)S_{\Xi}' = S_{\Xi} \cup A(\tau)SΞ′​=SΞ​∪A(τ), a pure set operation that cannot fail. ∎

Theorem 2: Group Atomicity​

Statement: States in a group ggg are always activated/deactivated together.

Proof: By group constraint, ∀s1,s2∈g:(s1∈A(τ)  ⟺  s2∈A(τ))\forall s_1, s_2 \in g: (s_1 \in A(\tau) \iff s_2 \in A(\tau))∀s1​,s2​∈g:(s1​∈A(τ)⟺s2​∈A(τ)). ∎

Theorem 3: Multi-Target Optimality​

Statement: The multi-target pathfinding algorithm finds globally optimal paths.

Proof: The algorithm explores all reachable state combinations using dynamic programming with memoization on (Sactive,Treached)(S_{active}, T_{reached})(Sactive​,Treached​) pairs, guaranteeing optimal solutions. ∎

Theorem 4: Occlusion Transitivity​

Statement: If s1s_1s1​ occludes s2s_2s2​ and s2s_2s2​ occludes s3s_3s3​, then s1s_1s1​ occludes s3s_3s3​.

Proof: Given ω(s1,s2,t)>θ\omega(s_1, s_2, t) > \thetaω(s1​,s2​,t)>θ and ω(s2,s3,t)>θ\omega(s_2, s_3, t) > \thetaω(s2​,s3​,t)>θ, by transitivity of visibility blocking, ω(s1,s3,t)>θ\omega(s_1, s_3, t) > \thetaω(s1​,s3​,t)>θ. ∎

Theorem 5: Safe Rollback​

Statement: Failed incoming transitions preserve system state.

Proof: Rollback restores SΞ,rollback=SΞ,initialS_{\Xi,rollback} = S_{\Xi,initial}SΞ,rollback​=SΞ,initial​, and EXIT phase is skipped on failure, preserving original state. ∎
